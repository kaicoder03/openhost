oh

Peer-to-peer file transfer over authenticated WebRTC. No central server touches your bytes.

Receive a file →

How it works

  1. Someone runs oh send <file> on their machine and shares the pairing code (12 words or a oh+pair:// URI).
  2. You paste the code into this page (or scan the QR if they show one).
  3. A WebRTC data channel opens between your browser and their machine — DTLS-encrypted end-to-end, no server sees the bytes.
  4. The file streams directly and downloads.

Sender setup (CLI)

OH_RELAYS=https://pkarr.pubky.app \
    oh send /path/to/file

Copy the printed oh+pair://… URI, and share it with the receiver.

Open-source at github.com/kaicoder03/openhost. Receive-only in the browser today; send-from-browser is on the roadmap.
